================================================================================

You are a senior SEO + Astro engineer. Build a single-page static website
optimized to rank #1 in Google for the PRIMARY_KEYWORD across every locale
in LANGUAGES. Output must be production-ready, deployable to Cloudflare Pages.


================================================================================

================================================================================

- Astro 4+ (Static Output, `output: "static"`).

Hreflang cluster (EVERY locale page must list EVERY locale + x-default):

44. ... (ko, de, fr, es, pt, it, ar, nl, sv, no, hi, ru as applicable)
45. <link rel="alternate" hreflang="x-default" href="https://{DOMAIN}/">

JSON-LD (one <script type="application/ld+json"> per page, an @graph
containing ALL of the following — this is more than the original sites ship,
which is part of how we beat them):

46. Organization        — name, url=PUBLISHER_URL, sameAs:[GITHUB_OR_PROJECT_URL, social links], logo
47. WebSite             — url, name, inLanguage={locale},
                          potentialAction: SearchAction with EntryPoint
                          (urlTemplate=https://{DOMAIN}/?q={search_term_string})
48. WebPage             — name, url=self, inLanguage, isPartOf=WebSite,
                          about=SoftwareApplication, primaryImageOfPage=og image
49. SoftwareApplication — name, applicationCategory="DeveloperApplication",
                          operatingSystem=PLATFORMS, offers={price:0, priceCurrency:"USD"},
                          license=LICENSE URL, author=Organization,
                          aggregateRating if you have real ratings (NEVER fake)
50. BreadcrumbList      — even on home page (Home > {Brand})
51. FAQPage             — wraps the FAQ section (6-8 Q&A pairs per locale)
                          Each item: Question { name, acceptedAnswer: Answer { text } }
52. HowTo               — wraps the Quick Start section
                          name, totalTime, step:[HowToStep x3], tool, supply (optional)

Critical: every text field in JSON-LD must be translated per locale.
@graph nodes must use absolute https://{DOMAIN}/... @id values with #anchors
(e.g. #organization, #website, #webpage) so Google can de-duplicate entities.

Hard rules:
- Exactly ONE <h1>, and it contains the PRIMARY_KEYWORD.
- 6-8 <h2>; every <h2> contains the brand or a topical keyword.
- 20+ <h3>; each H3 is a long-tail keyword candidate.
- Total visible body copy: 900-1200 words on EN, 300-600 on other locales (fine — they have ~no competition).
- Every <img> must have a meaningful alt="" describing the image + keyword.
- Every external link uses rel="noopener noreferrer" and target="_blank".
- Internal anchor links use #ids that match the section ids above.
- Use semantic HTML5: <header>, <main>, <section>, <article>, <footer>.

Translation rules:
- DO NOT machine-translate metaTitle, metaDescription, metaKeywords, or faqs.
  Use a native speaker / quality model and adapt to local search behavior.
- Keywords must be locally researched (e.g. zh: 开源AI智能体, ja: オープンソースAIエージェント).
- Titles 50-60 chars (or natural CJK length for zh/ja/ko/zh-tw).
- Descriptions 140-160 chars.
- Arabic file sets "dir": "rtl"; layout must flip.

<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9"
        xmlns:xhtml="http://www.w3.org/1999/xhtml">
  <url>
    <loc>https://{DOMAIN}/</loc>
    <lastmod>{YYYY-MM-DD}</lastmod>
    <changefreq>weekly</changefreq>
    <priority>1.0</priority>
    <!-- also emit xhtml:link rel="alternate" hreflang for every other locale -->

  ... one <url> per locale ...
  <url>
    <loc>https://{DOMAIN}/privacy/</loc>
    <changefreq>monthly</changefreq>
    <priority>0.5</priority>
  </url>
  <url>
    <loc>https://{DOMAIN}/terms/</loc>
    <changefreq>monthly</changefreq>
    <priority>0.5</priority>
  </url>
</urlset>
